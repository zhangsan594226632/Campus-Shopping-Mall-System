## 摘要

针对高校校园线下交易场景分散、学生购物渠道单一、校园商户缺乏线上经营载体、商品交易、订单售后、校园交流无统一数字化平台等痛点，本项目采用**SpringBoot后端\+Vue前端\+MySQL数据库**开发校园商城系统。系统采用前后端分离B/S架构，划分**三大角色**：平台超级管理员、入驻校园商家、在校学生用户，拆分为**后台管理系统**与**前台学生商城页面**两大业务板块。

系统完整覆盖商家入驻审核、商品上架管理、商品分类维护、在线购物车、下单支付、订单全流程管理、商品收藏点赞、校园论坛交流、系统公告、在线客服、轮播图配置、用户信息管控等核心电商业务，同时配套完整图片上传、多条件复合检索、分页、富文本编辑、角色权限隔离等通用功能。本系统实现校园商品线上数字化交易，统一整合校园商户资源，搭建校内师生交流论坛，规范校园二手/日用商品交易流程，具备极高校园实用价值。


# 第一章 绪论

## 1\.1 项目研究背景

高校校园内存在大量日用百货、零食、文创、二手商品交易需求，但传统线下交易模式存在显著弊端：

1. **交易渠道零散**：学生依靠线下摆摊、微信群私域交易，商品信息不透明、无统一展示平台；

2. **商户经营无线上载体**：校园小型商铺、个体摊主缺少线上店铺，无法线上展示商品、接单；

3. **交易流程不规范**：线下现金/私发转账无订单凭证，出现售后纠纷无追溯依据；

4. **缺少校园交流渠道**：学生无法线上交流购物体验、闲置物品置换、校园问题咨询；

5. **校方难以统一管控**：线下交易商品质量、经营主体无审核，易出现劣质商品、违规经营；

6. **数据无法统计**：无线上平台统计热销商品、交易订单、商户经营数据，缺少运营参考依据。

基于以上校园场景痛点，开发一套面向高校的校园线上商城系统，整合商户入驻、商品线上售卖、学生在线采购、校园论坛互动、平台统一监管全流程，实现校园商业数字化规范化管理，解决校园线下交易乱象，为师生提供安全便捷的校内线上购物平台。

## 1\.2 项目研究意义

### 1\.2\.1 平台管理员（校方运营）

1. 一站式管控全平台数据：管理学生用户、入驻商家、商品、订单、论坛帖子、公告、首页轮播图；

2. 商户资质审核机制：商家注册需上传营业执照、店铺资料，管理员审核后方可开店经营，过滤违规商户；

3. 商品分类、公告分类统一字典管理，灵活拓展商品类目、资讯类型；

4. 监管论坛内容，删除违规发帖、不良评论，维护校园文明交流环境；

5. 在线客服后台对接，接收学生购物咨询、投诉反馈；

6. 全量订单数据查看，统计校园商品交易规模、热门品类，支撑校园商业管理决策。

### 1\.2\.2 入驻校园商家

1. 线上自主开店：注册提交店铺资料，管理员审核通过后可上架、编辑、下架自有商品；

2. 完整商品管理功能：填写商品图文、规格、品牌、库存、单价、生产信息，实时修改库存；

3. 独立订单管理后台：查看本店全部学生下单订单，跟进订单发货、售后流程；

4. 店铺资料自主维护，修改店铺简介、营业时间、经营范围；

5. 查看商品点击、收藏、评论数据，分析商品受欢迎程度，调整经营品类。

### 1\.2\.3 在校学生用户

1. 线上浏览全校园商户商品，按商品名称、类型、品牌、价格区间多维度筛选；

2. 商品详情页支持点赞、收藏，直观查看商品实拍、规格、库存、商家信息；

3. 购物车批量选购，支持单件立即购买，填写收货地址生成订单；

4. 个人中心统一管理全部订单、收藏商品、收货地址、个人资料；

5. 校园论坛板块自由发帖、浏览、回复，交流闲置置换、购物心得、校园问题；

6. 浏览平台官方公告，查看校园活动、购物通知；

7. 在线客服实时咨询商品、订单相关问题。

### 1\.2\.4 行业与校园建设意义

本系统打造专属高校的轻量化校内电商平台，规范校园商品交易秩序，降低线下摆摊、私域交易带来的管理风险；打通商户与学生线上交易通道，方便学生日常采购，拓宽校园商户客源；配套校园论坛构建校内线上交流社区，完善校园数字化服务体系，符合智慧校园建设发展方向。

## 1\.3 系统开发软硬件环境与技术栈

### 1\.3\.1 核心开发技术栈

    - 核心框架：SpringBoot 2\.7，简化SSM配置，快速开发接口；

## 1\.4 系统开发目标

1. 实现**三类角色权限隔离**：平台管理员、入驻商家、学生用户，菜单、接口、操作权限严格区分；

2. 拆分两大独立业务端：后台管理系统（管理员\+商家双端后台）、前台学生商城页面；

4. 配套校园互动模块：论坛发帖留言、平台公告、在线客服；

5. 封装通用后台组件：多条件复合搜索、分页展示、单条/批量增删改查、多图上传、富文本编辑、表单必填校验；

6. 页面布局统一规范，操作逻辑简单易懂，适配主流PC浏览器，数据持久化存储，长期稳定运行无数据丢失。

# 第三章 系统需求分析

## 3\.1 系统角色划分

系统共三类核心使用角色，权限完全隔离、互不越权：

1. **平台超级管理员**：最高权限，管理全平台所有学生、商家、商品、订单、论坛、公告、系统字典，审核商家入驻资质，管控全平台违规内容；

2. **入驻校园商家**：店铺经营者，仅可管理自身店铺、本店商品、本店订单，维护店铺资料，无其他商家、全平台管控权限；

3. **在校学生用户**：前台商城使用者，浏览商品、下单购物、收藏点赞、论坛发帖、管理个人订单与地址，无任何后台管理权限。

## 3\.2 功能需求总览（结合页面截图梳理全模块）

系统分为**后台管理端**（管理员后台\+商家后台共用一套框架，菜单权限区分）、**前台学生商城端**两大板块。

### 3\.2\.1 后台管理端全部功能模块

1. **用户管理（管理员专属）**：管理所有注册学生账号，查询、修改、禁用、删除学生；

2. **商家管理（管理员专属）**：查看全部入驻商家，审核商家注册资质，禁用违规店铺；

3. **商品类型管理（管理员专属）**：维护商品一级分类字典，新增、删除、修改商品类目；

4. **商品信息管理（管理员\+商家）**

    - 管理员：查看全平台所有商品，下架违规商品、查看全部商品评论；

    - 商家：仅查看、新增、编辑、删除自家店铺商品，填写商品完整参数、上传商品图片；

5. **论坛交流管理（管理员专属）**：查看全平台学生发帖、留言，删除违规帖子、回复咨询；

6. **订单管理（管理员\+商家）**

    - 管理员：查看平台全部交易订单，筛选订单状态；

    - 商家：仅查看本店学生下单订单，处理发货、售后；

7. **系统管理（管理员专属，包含4个子模块）**

    - 公告信息分类：维护公告资讯分类字典；

    - 公告信息：发布校园平台公告、编辑删除公告；

    - 轮播图管理：配置前台首页顶部轮播宣传图；

    - 在线客服：后台对接学生咨询消息；

8. **我的资料（管理员\+商家）**：修改当前登录账号密码、个人基础信息。

### 3\.2\.2 前台学生商城端全部功能模块

1. **首页**：展示首页轮播图、平台公告、热门商品入口；

2. **商家**：浏览全部入驻校园店铺，查看店铺简介、营业时间、店内商品；

3. **商品信息**：多条件筛选全部商品，查看商品详情、点赞、收藏、加入购物车、立即购买；

4. **论坛交流**：查看全部学生帖子，按标题检索，自主发布新帖、回复他人留言；

5. **公告信息**：浏览平台发布的校园通知、活动公告；

6. **购物车**：管理已加入购物车商品，修改购买数量、批量结算；

7. **个人中心**：修改学生个人资料、密码、收货地址；查看我的订单、我的收藏、我的评价；

8. **在线客服**：实时发起咨询，反馈商品、订单问题。

## 3\.3 非功能需求

### 3\.3\.1 性能需求

1. 所有列表页面支持分页，默认每页10条数据，支持自定义每页展示条数；

2. 多条件组合模糊查询（商品名称\+类型\+品牌\+价格区间）响应时间≤1\.5s；

3. 图片上传支持2M以内图片，支持预览、裁剪、无卡顿保存；

4. 支持千级商品、订单、用户数据稳定加载，页面无崩溃、卡死。

### 3\.3\.2 安全需求

1. 登录身份拦截：未登录用户禁止访问个人中心、订单、购物车页面；

2. 角色权限拦截：商家只能操作本店数据，学生无法访问后台，管理员接口做身份校验；

3. 用户密码MD5加密存储，数据库不存储明文密码；

4. 表单输入校验：拦截空值、超长文本、非法图片格式，防止脏数据入库；

5. 商家入驻资质审核机制，未审核商家无法上架商品。

### 3\.3\.3 兼容性需求

前台、后台页面适配Chrome、Edge、360等主流PC浏览器，页面布局无错乱，按钮、上传、筛选、富文本功能全部正常。

### 3\.3\.4 可维护性需求

1. 商品类型、公告分类统一存放于系统管理模块，新增分类无需修改业务代码；

2. 后台所有列表、新增编辑页面复用通用组件，新增业务模块可直接复用查询、分页、上传逻辑。

# 第四章 系统总体设计

## 4\.1 系统整体分层架构（SpringBoot标准四层架构）

1. **表现层 View**
分为前台学生Vue页面、后台LayUI管理页面，接收用户表单输入、图片上传请求，展示商品、订单、论坛、公告数据；

2. **控制层 Controller**
SpringMVC控制器，接收前端HTTP请求，完成登录拦截、参数校验、角色权限判断，调用Service业务层，返回JSON数据或页面跳转；

3. **业务逻辑层 Service**
核心业务处理层，封装商家入驻、商品上架、下单结算、订单流转、论坛发帖、收藏点赞、资质审核等业务规则，处理多表关联逻辑，使用事务保证订单数据一致性；

4. **数据访问层 Mapper**
MyBatis\-Plus映射层，封装MySQL增删改查SQL，实现数据库数据读写；

5. **持久层 MySQL数据库**
存储管理员、商家、学生、商品、商品分类、订单、购物车、收藏、评论、论坛帖子、公告、轮播图、收货地址全部业务数据。

## 4\.2 系统功能模块结构说明

系统整体分为两大端：

1. **后台管理端**：面向平台管理员、入驻商家，包含用户、商家、商品、订单、论坛、系统字典、个人资料7大管理模块；

2. **前台学生商城端**：面向在校学生，提供首页、商家、商品、论坛、公告、购物车、个人中心、在线客服8大展示交互模块。

## 4\.3 角色权限管控设计表

|角色|可访问模块|操作权限限制|
|---|---|---|
|平台管理员|全部后台菜单、全平台所有业务数据|无操作限制，可新增/删除/修改任意商品、订单、用户、帖子，审核商家资质|
|入驻商家|商品信息、本店订单、我的资料|仅操作自身店铺商品、本店学生订单，无法查看其他商家数据、全平台用户|
|学生用户|前台全部商城页面、个人中心|仅浏览公开商品、操作本人收藏/订单/发帖，禁止访问任何后台管理页面|

## 4\.4 核心业务流程设计

### 4\.4\.1 商家入驻开店流程

1. 登录页面点击「注册商家」，进入商家注册表单；

2. 填写店铺名称、密码、店铺地址、经营范围、营业时间、联系电话；上传营业执照、店铺实拍图；填写富文本店铺简介；

3. 提交注册申请，等待平台管理员后台审核；

4. 管理员审核通过后，商家使用账号密码登录商家后台；

5. 商家进入商品信息管理，新增商品，填写商品名称、类型、品牌、规格、生产厂家、生产日期、价格、库存，上传商品图片，提交上架；

6. 商品自动同步至前台商城商品列表，学生可浏览、下单。

### 4\.4\.2 学生购物完整流程

1. 学生前台注册账号，填写账号、密码、姓名、性别、电话、头像完成注册；

2. 登录前台首页，浏览公告、热门商品；进入商品信息页面，按类型、品牌、价格筛选商品；

3. 进入商品详情页，查看图文参数，点击点赞、收藏；选择购买数量，添加购物车 / 立即购买；

4. 跳转确认下单页面，新增/选择收货地址，填写订单备注，提交订单；

5. 订单生成，商家后台收到订单，确认发货；

6. 学生个人中心查看订单物流状态，收货完成后可发表商品评价；

7. 闲暇时进入论坛交流板块，浏览帖子、发布校园相关分享。

### 4\.4\.3 管理员平台管控流程

1. 管理员登录后台，维护系统基础字典：商品类型、公告分类、首页轮播图；

2. 审核新提交的商家入驻申请，拒绝/通过开店资质；

3. 管控学生账号、商家店铺，禁用违规账号；

4. 查看全平台商品，下架违规、劣质商品；

5. 管理论坛帖子，删除不良内容，回复学生咨询；

6. 发布校园公告，维护前台首页宣传素材；

7. 查看全平台所有订单，统计校园交易数据。

# 第五章 数据库详细设计

## 5\.1 数据库设计规范

1. 数据库名称：`campus_shop`；

2. 字符集：utf8mb4，兼容中文、图片路径、特殊符号、emoji；

# 第六章 系统各模块详细功能设计（结合页面截图逐页说明）

## 6\.1 登录页面模块

### 页面功能说明

登录页面星空山体背景，居中登录表单：

1. 输入项：用户名、登录密码；

2. 下拉选择角色：管理员 / 商家；

3. 功能按钮：注册商家、登录；

4. 校验逻辑：空输入、账号不存在、密码错误、角色不匹配均弹窗提示；

5. 登录成功按角色自动跳转对应后台页面：管理员进入全功能后台，商家进入商家专属后台。

## 6\.2 注册页面模块（两类注册）

### 6\.2\.1 商家注册页面

表单字段全部带\*必填校验：店铺名称、密码、确认密码、店铺地址、经营范围、营业时间、联系电话；上传营业执照图片、店铺照片；富文本编辑器填写店铺简介；提交后进入管理员审核队列。

### 6\.2\.2 学生用户注册页面

必填项：用户账号、密码、确认密码、用户姓名；下拉选择性别；填写联系电话；上传头像图片；注册完成直接跳转前台商城首页。

## 6\.3 平台管理员后台各模块详情

后台统一布局：左侧固定菜单栏、顶部系统标题栏、右侧面包屑导航\+业务操作区，所有列表统一支持**搜索、新增、批量删除、分页、查看/编辑/删除单行数据**。

# 第八章 系统开发难点与解决方案

## 8\.1 难点1：三类角色前后端权限隔离，防止越权访问

**问题**：管理员、商家、学生权限边界清晰，商家只能操作本店数据，学生禁止访问后台，容易出现越权接口漏洞。
**解决方案**：

1. 登录时将角色、用户ID存入Session，后端全局拦截器统一校验；

2. 拦截器区分管理员/商家/学生身份，无权限接口直接跳转登录页；

3. 前端根据登录角色动态渲染菜单栏，商家隐藏全平台数据菜单，学生隐藏所有后台入口，前后端双重校验权限。

## 8\.2 难点2：多模块图片统一上传（商品图、营业执照、店铺图、头像、帖子配图）

**问题**：多业务模块图片上传，易出现同名图片覆盖、路径混乱、图片丢失。
**解决方案**：

1. 全局统一文件上传根目录，按业务创建独立子文件夹（goods、shop、user、forum）；

2. 上传时使用UUID随机重命名图片，杜绝同名覆盖；

3. 数据库仅存储图片相对路径，页面统一拼接静态资源域名渲染图片。

## 8\.3 难点3：订单多表关联事务处理（订单、购物车、商品库存同步）

**问题**：学生下单时需要扣减商品库存、删除购物车记录、新增订单，多表操作易出现数据不一致（库存扣减失败但订单生成）。
**解决方案**：

1. Service层使用Spring声明式事务，下单整套操作统一开启事务，任意步骤异常自动回滚；

2. 库存扣减采用悲观锁查询，防止超卖；下单成功同步清空购物车对应商品。

## 8\.4 难点4：多条件动态复合筛选（商品名称\+类型\+品牌\+价格区间）

**问题**：多维度同时筛选商品，SQL拼接繁琐，容易查询失效、空数据。
**解决方案**：

1. MyBatis动态SQL `<if>`标签判断前端传入参数，动态拼接WHERE查询条件；

2. 所有筛选参数为空时执行全量查询，有参数自动拼接过滤条件，支持价格区间区间筛选。

## 8\.5 难点5：商家店铺数据隔离，只能查看本店商品与订单

**问题**：多商家共用一张商品、订单表，商家登录后只能查询自身店铺数据，容易泄露其他商家信息。
**解决方案**：

1. 商家登录后Session存入shop\_id；

2. 商家后台所有商品、订单查询接口自动拼接`shop_id = 当前登录店铺ID`过滤条件，底层数据隔离，从根源杜绝越权查看。
